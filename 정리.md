## 학습 순서
** 1단계: Java (언어 기초)
* 목표: 스프링이 작성된 언어인 자바의 문법을 익히고, 객체지향의 개념을 잡는 단계.

* 기본 문법: 변수, 데이터 타입(int, String), 조건문(if, switch), 반복문(for, while)

* 객체 지향 (OOP) ⭐:

* 클래스와 객체 (new 키워드)

* 인터페이스 (Interface) (스프링의 핵심! 반드시 이해해야 함)

* 상속 (extends)과 다형성

* 컬렉션 프레임워크: List (ArrayList), Map (HashMap) 사용법

* 모던 자바 (Java 8+):

* 람다식 (Lambda)

* 스트림 (Stream) API (데이터 필터링/가공)

* Optional (Null 처리)

* 예외 처리: try-catch, throw

✅ Checkpoint: 자바로 간단한 계산기나 자판기 프로그램을 클래스를 나눠서 짤 수 있나요?

** 2단계: Web & DB 기초 (환경 이해)
* 목표: 스프링이 동작하는 웹 환경(HTTP)과 데이터를 저장할 창고(DB)를 이해하는 단계.

* Web (HTTP):

* 요청 메서드: GET(조회), POST(등록), PUT(수정), DELETE(삭제)

* 상태 코드: 200(성공), 400(클라이언트 잘못), 404(없음), 500(서버 터짐)

* 데이터 포맷: JSON 구조 이해하기 ({"name": "홍길동"})

* DB (SQL):

* 기본 쿼리: SELECT, INSERT, UPDATE, DELETE (CRUD)

* 조건절: WHERE, LIKE, ORDER BY

* 조인 (JOIN): 테이블 2개를 합치는 방법 (Inner, Left Join)

* 기본 개념: PK (기본키), FK (외래키), Transaction (트랜잭션)

✅ Checkpoint: 게시판 테이블을 만들고, SQL로 데이터를 넣고 빼고 수정할 수 있나요?

** 3단계: Spring Boot 입문 (프레임워크)
* 목표: 실제로 웹 사이트(API)를 만들어보는 단계. 가장 재미있는 시기.

* 프로젝트 설정: start.spring.io 사용법, Gradle 빌드 도구

* 스프링 핵심 원리:

* DI (의존성 주입): @Autowired, 생성자 주입

* IoC (제어의 역전): 빈(Bean)과 스프링 컨테이너

* 3계층 구조 (Layered Architecture) ⭐:

* Controller: 요청 받기 (@GetMapping, @RestController)

* Service: 비즈니스 로직 (실제 일하는 곳)

* Repository: DB 접근 (일단은 메모리나 JDBC로 시작)

* 편의 도구: Lombok (@Getter, @RequiredArgsConstructor)

* 테스트: JUnit5를 이용한 단위 테스트 작성법

✅ Checkpoint: 포스트맨(Postman)으로 요청을 보냈을 때 "Hello World"나 간단한 JSON 응답을 내려줄 수 있나요?

** 4단계: JPA (데이터 접근 기술)
* 목표: SQL을 직접 짜는 고통에서 해방되고, 객체지향적으로 DB를 다루는 단계.

* 기본 매핑:

* @Entity, @Id, @Column (테이블과 클래스 연결)

* JpaRepository 인터페이스 상속

* 쿼리 메서드: findByName, findByEmail 등 이름으로 조회하기 (이전에 질문하신 내용!)

* 연관관계 매핑 (가장 어려움) ⭐:

* @OneToMany, @ManyToOne (1:N, N:1 관계)

* 양방향 매핑과 연관관계 주인(Owner) 개념

* 고급 기능:

* 영속성 컨텍스트 (1차 캐시, 변경 감지/Dirty Checking)

* 지연 로딩 (Lazy Loading) vs 즉시 로딩 (Eager)

* 페이징 (Page, Pageable)

✅ Checkpoint: SQL 없이 자바 코드만으로 회원가입을 시키고, 작성한 게시글을 불러올 수 있나요?

** 5단계: Spring Security (보안)
* 목표: 아무나 들어오지 못하게 막고, 로그인 기능을 완성하는 단계.

* 기본 개념: 인증(Authentication) vs 인가(Authorization)

* 비밀번호 암호화: BCryptPasswordEncoder (해시 함수)

* 필터 체인: SecurityFilterChain 설정 (.permitAll(), .authenticated())

* 로그인 구현:

* UserDetailsService 구현하여 DB 회원 정보 연동

* JWT & OAuth2 (심화):

* JWT 토큰 발급 및 검증 필터 만들기

* 카카오/구글 소셜 로그인 연동 (OAuth2Client)

✅ Checkpoint: 로그인한 사람만 글을 쓸 수 있고, 로그인 안 한 사람은 튕겨내도록 만들 수 있나요?
